Thumb Simulator

                    decode and execute ARM Thumb instructions

In my computer architecture we were tasked to build an ARM Thumb simulator. We were given the base code of the simulator and were tasked to implement several methods to enable the simulator to interpret and execute ARM thumb instructions. Our simulator is able to handle a subset of the ARM Thumb instruction set. The program takes in as input several lines of instruction represented in binary and outputs the ARM instructions translated from the binary code.

Each type of instruction has a different procedure to decoding as different bits are used uniquely based on the instruction type. For this reason, we used the official ARM v7 Reference manual to figure out how to decipher each instruction type. For each line of binary instruction, we decode by associating their opcodes with their string equivalent and print out their instruction and registers used. After each instruction is decoded, next we fetch that instruction and update the registers used, data memory, and the program counter. This process is repeated for each instruction of the inputted file.

The Instructions that are supported included including push, pop, sub(sp minus immediate), sub(immediate), sub(register), add(sp plus register), add(sp plus immediate), add(register), add(immediate), cmp(register), cmp(immediate), mov(immediate), mov(register), str, strb, stmia, ldr, ldrb, ldr(literal), ldmia, b(unconditional), bcc, bcs, beq, bge, bhi, ble, bls, blt, bne, bl, lsl(immediate), neg(also called rsb).

After executing each line of instruction, we then collected several runtime statistics on the instruction set and the cache performance. For cache performance, we were using a 256-byte direct-mapped cache with several different block sizes to determine which one has the greatest hit rate.

Completing this project gave me and my group a better understanding how CPUs decode and execute instructions and a deeper understanding on what computer architects have to keep in mind when designing instruction sets.
